1. สรุปภาพรวม

BrainScope AI เป็นระบบวิเคราะห์ภาพถ่ายทางการแพทย์แบบครบวงจรที่ทำงานในเบราว์เซอร์ สามารถตรวจจับและจำแนกโรคทางสมอง 7 ประเภทจากภาพ CT/MRI ได้โดยอัตโนมัติ ระบบผสานโมเดล Object Detection บนคลาวด์ (Roboflow RF-DETR) เว็บไซต์บน GitHub Pages และระบบ Agent อัตโนมัติหลายตัวทำงานร่วมกัน

ความสามารถหลัก:

  • ตรวจหลายโรคพร้อมกัน พร้อมแสดงกรอบตำแหน่ง (Bounding Box) บนภาพ
  • รองรับไฟล์ DICOM (.dcm) โดยตรง — อ่านไฟล์จากโรงพยาบาลได้ทันทีในเบราว์เซอร์
  • สร้างรายงานรังสีวิทยาด้วย AI พร้อมระดับความเร่งด่วนและคำแนะนำส่งต่อผู้เชี่ยวชาญ
  • ระบบปรับปรุงตัวเองอัตโนมัติ — Agent จัดการรวบรวมข้อมูล ฝึกโมเดล และเผยแพร่ให้อัตโนมัติ

การประมวลผล CLAHE

Contrast Limited Adaptive Histogram Equalization (CLAHE) ถูกนำมาใช้กับภาพฝึกทุกใบก่อนอัปโหลด ต่างจาก Histogram Equalization แบบทั่วไป CLAHE ทำงานบนกระเบื้องภาพขนาดเล็ก ป้องกันการขยายสัญญาณรบกวนในบริเวณที่สม่ำเสมอ — สำคัญมากสำหรับ CT Scan ที่คอนทราสต์เนื้อเยื่อมีความละเอียดสูง

clahe = cv2.createCLAHE(clipLimit=4.0, tileGridSize=(8, 8))
enhanced = clahe.apply(grayscale_image)

การเรนเดอร์ Window/Level ของ DICOM

CT Scan เก็บค่าพิกเซลในหน่วย Hounsfield Units (HU) ระบบใช้ Brain Window (WC=40, WW=80) เป็นค่าเริ่มต้นหากไม่พบในส่วนหัว DICOM โดยแปลงค่า HU เป็นระดับสีเทา 0-255 ตามสูตร:

val = clamp(((HU - (WC - WW/2)) / WW) x 255, 0, 255)

ฐานข้อมูลโรคของ ReportWriter

โรคความเร่งด่วนการส่งต่อผู้เชี่ยวชาญ
bleeding (เลือดออกในสมอง)วิกฤตศัลยแพทย์ระบบประสาท (ฉุกเฉิน)
ischemia (สมองขาดเลือด)วิกฤตแพทย์ระบบประสาท / ทีม Stroke (ฉุกเฉิน)
glioma (เนื้องอก glioma)สูงศัลยแพทย์ระบบประสาท / มะเร็งระบบประสาท
meningioma (เยื่อหุ้มสมอง)ปานกลางศัลยแพทย์ระบบประสาท
pituitary (ต่อมใต้สมอง)ปานกลางศัลยแพทย์ระบบประสาท / แพทย์ต่อมไร้ท่อ
alzheimer (อัลไซเมอร์)ต่ำแพทย์ระบบประสาท / แพทย์ผู้สูงอายุ
normal (ปกติ)ไม่มีไม่จำเป็น

DataHunter — ตัวค้นหาข้อมูล

ค้นหาชุดข้อมูลภาพสมองจาก Roboflow Universe มากกว่า 7 แหล่ง ให้ความสำคัญกับคลาสที่มีข้อมูลน้อย (bleeding, ischemia) บันทึกผลลัพธ์ไว้ใน datahunter_results.json พร้อมจำนวนภาพและสถานะการเข้าถึง

รันด้วย: python agent_datahunter.py [--download]

AutoTrainer — ตัวฝึกและเผยแพร่อัตโนมัติ

เปรียบเทียบโมเดลที่ฝึกใหม่กับโมเดลที่ดีที่สุดในปัจจุบัน (บันทึกใน autotrainer_state.json) ทดสอบความแม่นยำบนภาพทดสอบท้องถิ่น หากโมเดลใหม่ดีกว่าจะอัปเดต frontend/index.html commit และ push ขึ้น GitHub Pages โดยอัตโนมัติ

รันด้วย: python agent_autotrainer.py [status|deploy|test N|check-v6]

Inspector — ตัวตรวจสอบคุณภาพ

ตรวจสอบคุณภาพชุดข้อมูล 3 ด้าน: (1) นับภาพต่อคลาสเพื่อตรวจสอบความสมดุล (2) คำนวณ Laplacian variance ตรวจจับภาพเบลอ (เกณฑ์ <50) (3) ใช้โมเดลปัจจุบันตรวจหาภาพที่ prediction ไม่ตรงกับ label ด้วยความมั่นใจ ≥50%

รันด้วย: python agent_inspector.py [full|balance|quality|mislabeled]

ReportWriter — ตัวสร้างรายงาน (ทำงานในเบราว์เซอร์)

JavaScript ที่ฝังอยู่ใน frontend/index.html ทำงานเมื่อกดปุ่ม "สร้างรายงาน" หลังวิเคราะห์ แปลงผลการตรวจจับเป็นรายงานทางการแพทย์โดยอ้างอิงจากฐานข้อมูล DISEASE_DB ที่มีระดับความเร่งด่วน คำแนะนำผู้เชี่ยวชาญ และการดำเนินการทางคลินิก

ผลลัพธ์: หน้าต่าง HTML พร้อมแบนเนอร์ความเร่งด่วน ตารางผลการตรวจ รายชื่อผู้เชี่ยวชาญ และข้อจำกัดความรับผิดชอบ

Python 3.11
pip install roboflow opencv-python-headless numpy requests pyyaml fastapi uvicorn python-dotenv
Node.js (ไม่บังคับ สำหรับ dev server ท้องถิ่น)

ขั้นตอนที่ 1: โคลน repo และตั้งค่า

git clone https://github.com/chokdee255120-dot/brainscope-ai.git
cd brainscope-ai
git config user.email "totoolclaude@gmail.com"
git config user.name "chokdee255120-dot"

ขั้นตอนที่ 2: รัน Backend ท้องถิ่น (ไม่บังคับ)

start_backend.bat           # Windows — เริ่ม FastAPI ที่ http://localhost:8000
# หรือ:
cd backend && uvicorn main:app --reload

ขั้นตอนที่ 3: เผยแพร่การเปลี่ยนแปลง Frontend

# แก้ไข frontend/index.html แล้ว:
Copy-Item frontend/index.html index.html    # PowerShell
git add index.html frontend/index.html
git commit -m "ข้อความ commit"
git push origin main
# GitHub Pages จะ rebuild อัตโนมัติ (~2 นาที)

ขั้นตอนที่ 4: ฝึกโมเดลเวอร์ชันใหม่

# แบบเร็ว: สร้าง version จาก 21,806 ภาพที่มีอยู่และฝึกเลย
python gen_v6_now.py

# แบบเต็ม: CLAHE preprocessing + upload + สร้าง version + ฝึก (ข้ามคืน)
python v6_build.py

# ติดตามและ deploy อัตโนมัติเมื่อฝึกเสร็จ
python watch_v6.py

ขั้นตอนที่ 5: เผยแพร่โมเดลที่ดีที่สุด

python agent_autotrainer.py status    # ดูโมเดลที่ดีที่สุดในปัจจุบัน
python agent_autotrainer.py deploy    # เปรียบเทียบ + deploy อัตโนมัติ

ขั้นตอนที่ 6: ตรวจสอบคุณภาพชุดข้อมูล

python agent_inspector.py full        # รายงานคุณภาพแบบเต็ม
python agent_datahunter.py --download # ค้นหา + ดาวน์โหลดข้อมูลใหม่

การตั้งค่า GitHub Pages

Settings → Pages → Source: Deploy from branch → main → / (root)

9. งานที่ค้างและแผนงานต่อไป

ลำดับความสำคัญงานผลกระทบที่คาดหวัง
วิกฤตDeploy v6 เมื่อฝึกเสร็จ → python agent_autotrainer.py deployประโยชน์จาก CLAHE + augmentation
สูงแก้ไข add_hemorrhage.py — ดาวน์โหลดชุดข้อมูล brain-hemorrhage (SDK issue รูปแบบ multilabel-class)+834 ภาพ bleeding → recall ดีขึ้น
สูงค้นหาและเพิ่มชุดข้อมูล CT ischemia/strokeเพิ่ม ischemia จาก ~113 เป็น 2,000+ ภาพ
ปานกลางรัน agent_inspector.py full → ตรวจสอบภาพที่ label ผิดชุดข้อมูลสะอาดขึ้น → precision สูงขึ้น
ปานกลางv7: ฝึกบนชุดข้อมูล CLAHE (v6_build.py กำลัง upload อยู่)ได้ประโยชน์จาก CLAHE เต็มที่
ต่ำอัปเกรดแผน Roboflow → เข้าถึง RF-DETR Large / accurate trainingmAP อาจขึ้นถึง 98%+
ต่ำเพิ่ม Confidence Calibration ใน ReportWriterรายงานความไม่แน่ใจได้ซื่อตรงขึ้น

แนวทางสู่ mAP 98%+

อ้างอิงจากงานวิจัยที่ตีพิมพ์ (YOLO11 พร้อม attention: 99.3% mAP@0.5, RF-DETR Large: 99.5% บน meningioma) การบรรลุ 98%+ ต้องการ:

  1. RF-DETR Large (ต้องใช้แผนชำระเงิน Roboflow — accurate training mode)
  2. ชุดข้อมูลสมดุล — อย่างน้อย 1,000 ภาพต่อคลาสสำหรับทั้ง 7 คลาส
  3. CLAHE preprocessing บนภาพฝึกทั้งหมด (v7 กำลัง upload อยู่)
  4. Full augmentation — mosaic, mixup, rotation, flip, brightness, noise
  5. label สะอาด — ลบ label0/1/2 ที่ปนเปื้อนผ่าน Roboflow class remapping
